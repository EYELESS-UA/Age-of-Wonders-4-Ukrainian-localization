# 🌿 Правила роботи з гілками (Branches)

Щоб розробка українізатора просувалася чітко, а оновлення гри чи DLC не ламали поточний переклад, у цьому проекті використовується адаптована модель **Git Flow**. 

Будь ласка, ознайомтеся з нашою структурою гілок перед тим, як починати роботу.

---

## 🏛️ Головні гілки (Постійні)

### 🔵 `main` (або `master`) — Стабільний реліз
* **Що тут лежить:** Тільки повністю готова, протестована та актуальна версія перекладу.
* **Для кого:** Звідси готові файли з папки `build` забирають гравці, а також платформи **LBK Launcher** та **Steam Workshop**.
* **⚠️ Правило:** Ніколи не робіть Pull Request (PR) у цю гілку напряму. Зміни сюди потрапляють лише з гілки `develop` під час релізу нової версії.

### 🟡 `develop` — Робоча чернетка (Гілка за замовчуванням)
* **Що тут лежить:** Найсвіжіші напрацювання, перекладені блоки, які ще проходять вичитку, та виправлені одруки. Текст тут може бути готовий не на 100%, але гра не повинна вилітати.
* **⚠️ Правило:** Усі ваші Pull Request'и мають бути спрямовані **саме в гілку `develop`**.

---

## 🛠️ Тимчасові гілки (Під конкретні завдання)

Якщо ви плануєте зробити велике оновлення або перекласти окремий масив тексту, **створюйте нову гілку від `develop`** та використовуйте відповідні префікси в назвах:

### 1. `feature/...` — Переклад нових великих блоків
Використовується для перекладу конкретних механік, сюжетів чи інтерфейсу.
* *Приклади:* `feature/tome-of-souls`, `feature/interface-menus`, `feature/hero-skills`.

### 2. `fix/...` — Виправлення помилок
Використовується для швидкого усунення технічних багів, критичних одруків чи зламаних XML-тегів.
* *Приклади:* `fix/xml-tag-error`, `fix/font-glitch`, `fix/typo-in-intro`.

### 3. `dlc/...` — Адаптація під нові патчі та доповнення
Створюється координатором проекту, коли Triumph Studios випускає оновлення. Тут оригінальні файли в `src/source/` оновлюються свіжим англійським текстом і ведеться робота над новими рядками, не заважаючи основному процесу розробки.
* *Приклад:* `dlc/ways-of-war`.

## 🚀 Покроковий алгоритм: як зробити свій внесок

1. Зробіть **Fork** цього репозиторію.
2. Перейдіть у свою локальну гілку `develop`.
3. Створіть нову гілку для свого завдання:
```bash
   git checkout -b feature/назва-вашого-завдання
